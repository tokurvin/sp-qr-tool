Code Format Reference — Current (v4)
v4 Format — URL-based · EU Battery Regulation compliant
HTTPS://SP.SWAPPIE.COM/POFI-00088285-SMAPLP1003-01-05-03-0010:BK
URL prefix — domain TBD; stripped by Hemuli & WMS before parsing
PO country + PO number — 2-letter country code + hyphen + 8 digits
SKU — SM/TB/LT + APLP + 1–5 digits
Shipment # · Total boxes · Box # · Qty — all hyphen-separated
Two check characters — colon-separated, A–Z + 0–9 + $
Algorithm C1 = hash(path, 31) mod 37
C2 = hash(path, 17) mod 37
Hash runs left to right over the path (after the URL prefix, before :).
Output alphabet (37 chars) A–Z + 0–9 + $
Two independent check characters detect all single-char errors and adjacent transpositions.
Character values 0–9 → 0–9   A–Z → 10–35   - → 36
URL prefix is excluded from check computation. Only the path data is hashed.
QR spec Version 4-Q · 25% error correction · fits 20×20 mm sticker
Domain length is constrained to ≤16 chars to stay within QR capacity.
Code Format Reference — v1 (currently in production · no tooling · phasing out)
v1 Format — no check digit · plain text · still scanned in Hemuli & NetSuite
1234-LTAPLP12345POFI-87654321-01-98-12
Batch qty — 4 digits, zero-padded
SKU — 9–11 chars, starts with letter (hyphen-separated from qty)
PO country + PO number — 2-letter code + 8 digits (no hyphen between them)
Shipment # · Total boxes · Box # — all 2-digit, hyphen-separated
No check character — no error detection
Validation regex ^\d{4}-[A-Z][A-Z0-9]{8,10}[A-Z]{2}\d{8}-\d{2}-\d{2}-\d{2}$
Status Placed on batteries by suppliers since rollout. Hemuli and NetSuite/WMS must continue to accept this format throughout the v4 transition. No new codes should be generated in this format.
