Code Format Reference
v3 format Production
0155SMAPLP1829POFI-00090005-01-15-01J
Batch qty — 4 digits, zero-padded
SKU — 9–11 chars
PO number — PO + 2 letters + hyphen + 8 digits
Shipment # · Total boxes · Box # — all 2-digit
Check letter — A–Z or $ % *
Algorithm acc = (acc × 31 + val(c)) mod 29
Rolling hash over all chars. Map result to output alphabet.
Output alphabet (29 chars · prime) A–Z + $ % *
100% detection of single-char errors and adjacent transpositions.
Character values 0–9 → 0–9   A–Z → 10–35   - → 36
QR spec Version 2 · ECC Medium (15%) · Alphanumeric · 38 char capacity

Two check chars — A–Z, 0–9, $
Two independent check chars Char 1: acc = (acc × 31 + val(c)) mod 37
Char 2: acc = (acc × 17 + val(c)) mod 37
Output alphabet (37 chars · prime) A–Z + 0–9 + $
100% single + double error detection; 99.93% for three+.
Computed over path only URL prefix excluded — v4.0 and v4.1 produce identical check chars for the same data.
