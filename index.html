Spare Parts QR Tools

Tools for generating, validating and registering battery data for spare parts. The v4.0 QR spec is the unified standard — same code serves both internal SP traceability and EU Battery Regulation 2023/1542 customer disclosure.

v1 In production No check char · QR V2-M v3 MOD 29 · 1 check char Deprecated v4.0 MOD 37 · 2 check chars URL QR · EU compliant Unified standard
2026-05-12 decision (SP traceability meeting): v4.0 is the unified standard going forward — the same code serves SP traceability (WMS / Hemuli scan the string as a lot identifier) and customer-facing EU Battery Regulation disclosure (phone camera opens the URL). v3 and v4.1 are deprecated — kept here for reference and for handling legacy codes still in the field.

Code Format Reference
v4.0 format — current standard Canonical
HTTPS://QR.SWAPPIE.COM/POFI-00090005-SMAPLP1829-01-15-01-0155:BK
URL prefix — opens browser directly when scanned by a phone
PO number
SKU
Shipment # · Total boxes · Box #
Batch qty — 4 digits, zero-padded
Two check chars — A–Z, 0–9, $
Dual use Internal scanners (WMS, Hemuli) read the string as a flat lot identifier — no browser, no URL fetch. Customer phones read the URL and open the EU data sheet.
Two independent check chars c1 = hash(str, mult=31) mod 37
c2 = hash(str, mult=17) mod 37
Output alphabet (37 chars · prime) A–Z + 0–9 + $ — catches single-char errors, transpositions, and most multi-char corruptions.
QR symbol Version 4-Q (33×33 modules, 25% error correction). Fits on a 20×20mm sticker.
v1 format — current production legacy Phasing out
1234LTAPLP98765POFI-1234567899-11
Batch qty — 1–5 digits
SKU — starts with SM/TB/LT + APLP + digits
PO number — PO + 2 letters + hyphen + 8 digits
Total boxes (up to 4 digits) · Box # (up to 4 digits)
No check character — no error detection
